# 論文道場 入力ルール・UI設計方針

## 概要

論文道場（`dojo.html`）のStage 0入力フォームは、初心者が途中で詰まらないよう「**Q1のみ必須、Q2〜Q5は任意・あいまい入力可**」という方針で設計されている。

---

## 入力必須ルール

| セクション | 番号 | 必須 / 任意 | 備考 |
|-----------|------|-------------|------|
| 著者情報 | Q1 | **必須** | 著者氏名（`authorName`）が1文字以上あれば分析開始ボタンを解放 |
| 論文情報 | Q2 | 任意 | 空欄・あいまい入力でも可。論文題名のみ `*` 表示を残す（UI上の目安） |
| 要旨・研究課題 | Q3 | 任意 | メモ書き程度でも可。空欄でもブロックしない |
| 論文構成 | Q4 | 任意 | デフォルトの章立てが初期入力済みのため実質ブロックしない |
| 先行研究・参考文献 | Q5 | 任意 | 0件でも先へ進める。6件以上で満点 |

---

## スコアロジック（`calcScore()`関数）

スコアは最大30点。各軸5点満点。

### Q1 著者情報（必須）

```js
const authScore = (auth && email && aff) ? 5
                : (auth && (email || aff)) ? 3
                : auth ? 1
                : 0;
```

### Q2〜Q5（緩和スコア）

```js
// Q3 研究課題 RQ
const rqScore = rq.length >= 20 ? 5 : rq.length >= 10 ? 3 : rq.length > 0 ? 1 : 0;

// Q3 要旨
const absScore = abs.length >= 200 ? 5 : abs.length >= 50 ? 3 : abs.length > 0 ? 1 : 0;

// Q2 投稿フォーマット（未選択でも1点付与してブロックしない）
const fmtScore = fmt ? 5 : 1;

// Q5 参考文献
const refScore = refs >= 6 ? 5 : refs >= 3 ? 3 : refs > 0 ? 1 : 0;

// Q4 章立て（デフォルト17章が入力済みのため常に満点相当）
const structScore = toc >= 5 ? 5 : toc >= 2 ? 3 : toc > 0 ? 1 : 0;
```

## 分析開始ボタンの解放条件

```js
const q1Ready = auth.length > 0;  // 著者氏名1文字以上

if (q1Ready) {
  btn.disabled = false;
  btn.classList.add('ready');
  // Q2〜Q5が薄い場合はヒントを表示（ブロックはしない）
}
```

**旧ルール（廃止）：スコア28点以上でないと解放しない** → 廃止済み。

## アシストヒント表示仕様

### セクション冒頭：`.assist-hint` ブロック（青いボックス）

Q2〜Q5の各セクション冒頭に設置。メッセージ例：

- **Q2**：「Q2はあいまい入力・空欄でも先へ進めます。題名や投稿先が未定の場合は、仮のタイトルやキーワードだけでも入力しておくと…」
- **Q3**：「Q3はあいまいな言葉・断片的な文章でも大丈夫です。まだ書けない場合はメモ書き程度に入力するだけでも構いません。」
- **Q4**：「Q4はデフォルトの章立てが入力済みです。そのまま使っても、一部だけ書き換えても構いません。」
- **Q5**：「Q5は空欄のまま先へ進めます。6件以上入力するとスコアが満点になり、AIの引用提案精度も大幅に上がります。後でまとめて入力することをおすすめします。」

### フィールド下：`.assist-hint-inline`（小さな💡テキスト）

各フィールドの下に追加。`::before { content: "💡" }` で装飾。

### スコアバーのヒントテキスト（`#scoreHint`）

- Q1未入力時：「⚠️ 01 著者情報の「著者氏名」を入力すると分析開始ボタンが有効になります」
- Q1入力済み・Q2〜Q5が薄い場合：「✅ 分析開始できます。💡 後で入力すると精度が上がります：**〇〇・〇〇**」
- 全項目入力・28点以上：「✅ スコア28点以上です。「分析開始」を押してください。」

## ラベル表記ルール

- **Q1フィールド**：`<span class="req">*</span>`（赤いアスタリスク）を残す
- **Q2〜Q5フィールド**（必須解除済み）：`*` を削除し `<span class="opt-badge">任意</span>` に置き換える
- セレクトの初期値：「選択してください」→「未定 / あとで選ぶ」に変更

## クレジット制UI・ゼロストップ・ガバナンス（2026-05追加）

### 設計思想

「**ユーザーのデータ消失リスク**」と「**勝手な自動課金への恐怖**」を完全に払拭する「ゼロストップ・ガバナンス」UXを採用している。

---

### CSS変数・クラス一覧（クレジット関連）

| クラス名 | 用途 |
|---------|------|
| `.credit-meter-wrap` | スコアバー内のクレジットメーター全体ラッパー |
| `.credit-meter-label` | 「残りクレジット」ラベル行 |
| `.credit-free-badge` | 初回50cr付与バッジ（初回のみ表示） |
| `.credit-display` | クレジット数値（通常=`#F1C40F`、caution=オレンジ、warning=赤） |
| `.credit-display.caution` | 残量21〜10cr時の警告色（`color:#E67E22`） |
| `.credit-display.warning` | 残量10cr以下の危険色（`color:#E74C3C`） |
| `.credit-bar-bg / .credit-bar-fill` | バープログレス。fill も `.warning/.caution` クラスで色変化 |
| `.btn-charge` | 「＋ チャージ」ボタン（スコアバー内、ページ遷移なしでモーダルを開く） |
| `.zero-stop-note` | 「残量0でも自動引き落としは一切されません」の常時表示ノート |
| `.btn-analyze.credit-blocked` | クレジット不足時のdisabled赤色スタイル |

---

### クレジット定数・初期値

```js
const ANALYSIS_COST = 10;   // 分析1回あたりの消費クレジット
// 初回付与: 50cr（localStorage未設定時のデフォルト）
// 最大残高: 200cr（チャージ上限）
```

---

### updateCreditUI() の仕様

- `localStorage('dojo-credits')` から残量を読み込む
- 残量0でも自動リダイレクト・自動課金は一切行わない
- 残量 ≤ 10 → `.warning`（赤）
- 残量 ≤ 20 → `.caution`（オレンジ）
- 残量 < `ANALYSIS_COST` → `btn.disabled = true` + `.credit-blocked` クラス付与 + ボタンテキスト「クレジットが不足しています」
- `calcScore()` 末尾で必ず `updateCreditUI()` を呼ぶこと

---

### チャージモーダル（`#chargeModal`）仕様

#### 開閉
```js
openChargeModal()   // .modal-overlay に .open を追加
closeChargeModal()  // .open を削除
```

#### 設計原則（変更禁止事項）
1. **ページ遷移ゼロ**：チャージボタンを押してもページ移動しない（モーダル内完結）
2. **データ保護バナー必須**：`modal-data-safe` ブロックに「入力データはブラウザに自動保存されています」を常に表示
3. **ゼロストップ保証文必須**：`modal-zero-stop` ブロックに「自動引き落としは一切されません」を常に表示

#### 決済手段の制限（絶対ルール）

> **クレジットカード（Visa / Mastercard / Amex / JCB）のみ。**
> PayPay・楽天Pay等のローカル決済・QRコード決済・電子マネーは**一切導入しない（NOとする）**。

モーダル内の決済UIは以下の構成のみ許可：
- `.stripe-btn`：Stripe Checkoutへの遷移ボタン（紫 `#635BFF`）
- `.card-brands`：VISA / Mastercard / Amex / JCBのテキストのみ表示（ロゴ画像不使用）
- ロゴ画像・複数の決済アイコン・QRコードは**使用禁止**

#### プラン選択クラス

```js
// 選択状態は .selected クラスで管理（.featured は使用しない）
document.querySelectorAll('.plan-card').forEach(c => c.classList.remove('selected'));
el.classList.add('selected');
```
