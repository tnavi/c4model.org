【モニター参加者へのご案内】データの保存・再生について

本システムは外部のデータベースに個人情報や研究データを送信せず、お使いの端末（ブラウザ）内にのみ安全にデータを保存します。そのため、アカウント登録不要で全国どこからでも即座にテスト利用が可能です。
作業を中断・再開する際は、必ず下の「💾 下書き保存」を押し、次回アクセス時に「📂 下書き読込」を押すことで、すべてのプロンプトと入力状態を完全復元（再生）できます。

STAGE II

C4モデル構造化

4層（L1〜L4）で論文の骨格を設計します。

L1 · Context — 研究の世界観

「なぜ今この研究が必要か？」社会的背景・外部アクターを記述します。

L2 · Container — 章立て構造

論文の各章を「コンテナ」として配置し、論理の流れを可視化します。

L3 · Component — 論点・データ

各章の主要な論点・実験・引用を「コンポーネント」として整理します。

L4 · Code — 文・接続詞

接続詞・文末表現・引用番号の統一ルールを設定します。

【A】まだ修正が必要な場合（各Stageで再検討）

入力の抜け漏れや、論理の飛躍がある場合は、ここから直接各Stageに戻って入力データを修正してください。修正後、再度「🔄 プレビュー生成」を押すことでデータが再反映されます。

【B】このまま進める場合（Claudeで清書・仕上げ）

内容が十分に整っていると判断した場合は、「全文をコピー」し、すぐ下にある【清書・仕上げプロンプト】と一緒にClaudeへ送信して完成稿を作成させます。
