STAGE II · C4モデル構造化

論文の設計図を描く

C4モデルの4層（Context→Container→Component→Code）で論文構造を可視化します。

L1 · Context — 研究の世界観

「この研究はなぜ今必要か？」社会的背景・先行研究・外部要因を記述します。

以下をClaudeに送信してL1 Context図（PlantUML）を生成： --- あなたはC4モデルの専門家です。以下の研究情報からL1 Context図をPlantUML形式で生成してください。 論文題名：[Stage 0で入力した題名] 研究課題：[Stage 0で入力したRQ] 投稿先：[Stage 0で入力した学会] 出力形式： @startuml !include https://raw.githubusercontent.com/plantuml-stdlib/C4-PlantUML/master/C4_Context.puml title L1 Context — [論文題名] Person(researcher, "研究者", "著者") System(paper, "本研究", "研究目的") System_Ext(prior, "先行研究", "関連分野") Rel(researcher, paper, "執筆") Rel(paper, prior, "参照") @enduml --- C4モデルページで図を描画できます。

L2 · Container — 章立て構造

論文の各章を「コンテナ」として配置し、論理の流れを可視化します。

L3 · Component — 論点・データ

各章内の主要な論点・実験・引用を「コンポーネント」として整理します。

L4 · Code — 文・接続詞

接続詞・文末表現・引用番号の統一ルールを設定します。
